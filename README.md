# Heart disease exploration and prediction in R

Exploratory analysis and interpretable prediction using the UCI Heart Disease dataset in R.

## Project overview

This project uses the public UCI Cleveland Heart Disease dataset to explore clinical variables associated with heart disease and build a simple logistic regression model.
The dataset can be accessed here: https://archive.ics.uci.edu/ml/machine-learning-databases/heart-disease/processed.cleveland.data

The aim of this project is to demonstrate a clear R workflow including data cleaning, exploratory summaries, visualisation, unsupervised exploration, feature selection, model fitting and model evaluation.

## Dataset

The dataset used is the processed Cleveland Heart Disease dataset from the UCI Machine Learning Repository.

The original 'diagnosis' variable ranges from 0 to 4, where 0 indicates no heart disease and values 1–4 indicate heart disease diagnosis levels.

In this project, I used this variable in two ways:

- 'diagnosis' was kept for exploratory analysis of diagnosis level.
- 'disease_binary' was created for modelling, where:
  - '0' = no heart disease
  - '1' = any heart disease present, combining original diagnosis values 1–4.

Rows with missing values were removed before modelling.

## Pipeline

1. Load the dataset.
2. Check missing data and save a missing data summary.
3. Remove rows with missing values.
4. Create a binary heart disease outcome.
5. Convert categorical variables into factors.
6. Create exploratory summary tables.
7. Produce exploratory plots.
8. Use PCA to visualise the spread of numerical clinical variables.
9. Use k-means clustering as an exploratory unsupervised method.
10. Investigate the clinical composition of the clusters.
11. Select features based on exploratory analysis and clinical relevance.
12. Split the data into training and test sets.
13. Fit a logistic regression model.
14. Predict heart disease status on the test set.
15. Evaluate performance using accuracy, sensitivity and specificity.

## PCA and clustering exploration

PCA was used to visualise the spread of patients using selected numerical clinical variables: age, resting blood pressure, cholesterol, maximum heart rate and oldpeak.

K-means clustering was then used as an exploratory unsupervised method to assess whether patients formed broad groups based on these numerical variables. The clusters were compared with heart disease status and diagnosis level.

The clusters did not perfectly separate patients with and without heart disease, so they were not treated as diagnostic groups. However, cluster composition suggested that one cluster contained a higher proportion of no-disease patients, and among diseased patients in this cluster, diagnosis level 1 was the most common. This was interpreted as exploratory evidence of possible lower-severity structure in the data, rather than a clinically validated grouping.

## Model

A logistic regression model was fitted because the outcome was binary and the method is interpretable.

The model used the following predictors:

- age
- sex
- chest pain type
- maximum heart rate
- exercise-induced angina
- oldpeak
- number of major vessels
- thalassemia status

Resting blood pressure and cholesterol were explored but were not included in the final model to keep the model simple and interpretable.

The original 'diagnosis' variable was not used as a predictor because it was used to define the binary outcome.

## Brief interpretation

The exploratory plots showed overlap between patients with and without heart disease, so they were not treated as evidence of clear clinical groups. For example, 'pca_disease_plot.png' showed that disease and no-disease patients were not cleanly separated using the first two principal components of selected numerical clinical variables.

The k-means clustering plot, 'kmeans_pca_plot.png', suggested two broad groups in the numerical data. However, these clusters did not perfectly correspond to disease status, so they were not interpreted as diagnostic groups. I therefore explored the cluster composition further using 'diagnosis_composition_by_cluster.png' and 'diagnosis_composition_by_cluster.csv'.

This showed that cluster 2 contained a higher proportion of no-disease patients, and that the disease cases within this cluster were mostly milder diagnosis level 1 cases rather than higher diagnosis levels. This suggests the clustering may be capturing some lower-severity structure in the data, but this was treated as exploratory only.

For prediction, I therefore used the clinically interpretable binary outcome of no heart disease versus any heart disease present and fitted a logistic regression model using selected clinical predictors.

## Limitations of analysis

This is a small public dataset from a single source, so the model should not be used as a clinical prediction tool.

A single train/test split was used for simplicity. Future work could use repeated cross-validation and external validation to obtain a more robust estimate of model performance.

K-means clustering was used only as an exploratory method. The clusters should not be interpreted as clinically meaningful patient subgroups without further validation.

The purpose of this project is to demonstrate R coding, data cleaning, exploratory analysis, interpretable modelling and documentation of a reproducible workflow.
